## 개요

이 문서에서는 게임 속에서 사운드 시스템을 어떻게 구현할 것인지에 대해 설명한다.

## 사운드의 유형

- 모노 사운드: 오브젝트의 위치 구분 없이 일괄적으로 들리는 오디오

> [!info] 모노 사운드
> 오브젝트의 위치 구분 없이 일괄적으로 들리는 오디오.
> 
> 주로 UI 사운드, 캐릭터 피격 효과음, BGM 등 위치 구분이 필요 없는 사운드에 사용한다.
> 사운드 출력은 [[#사운드 매니저]]에서 담당한다.

> [!info] 공간 음향
> 단순히 좌우를 넘어 위, 아래, 왼쪽, 오른쪽 위치가 구분이 되는 음향.
> 
> 플레이어의 투사체 사운드 효과 등에 사용한다.
> 사운드 출력은 해당 사운드가 출력될 게임오브젝트에서 관리한다.

## 구현 해야 할 것들

> [!NOTE] 사운드의 '논리적 그룹'이란
> 사운드 종류의 분류.
> 각 논리적 그룹은 `SoundData` 스크립터블 오브젝트로 묶이며, `SoundManager`에서도 각각 서로 다른 `Audio Source` 컴포넌트와 연결된다.
> 
> - Master: 전체 볼륨 제어
> - BGM: 배경음악 (Loop), 페이드 효과 적용)
> - SFX: 효과음 (플레이어, 적, 환경음 등 3D 사운드 포함)
> - UI: 인터페이스 조작음

- 사운드 파일들을 논리적 그룹 단위로 묶을 스크립터블 오브젝트
- 사운드 출력할 `SoundManager`
- 위 콜아웃에 명시된 논리적 단위의 사운드 볼륨을 조절할 `AudioMixer`

## 스크립터블 오브젝트

사운드 데이터를 분류하기 위한 데이터를 논리적 묶음 단위로 분리하기 위한 스크립터블 오브젝트.
예를 들어, 하나의 스크립터블 오브젝트 스크립트를 기반으로 이런 스크립트 파일들을 제작한다:
- 플레이어 사운드
- 적 사운드
- BGM
- 무기 사운드

### 스크립터블 오브젝트 스크립트에서 구현해야 할 것

- 사운드 파일 저장하기 위한 변수
	- 파일 개수의 조절이 자유로워야 함
	- 따라서 배열을 사용할 예정
- 사운드 파일을 다른 클래스로 전달하기 위한 메서드

## 사운드 매니저

사운드 매니저는 싱글톤으로 구성되는 게임 오브젝트이다.
[[#사운드의 유형]]에서 언급한 **모노 사운드**의 출력을 담당한다.

사운드 출력 명령은 사운드 출력을 원하는 스크립트에서 사운드매니저의 함수를 선언하여 가능하다.

서로 다른 사운드는 서로 다른 `Audio Source` 컴포넌트를 사용한다.
각각의 오디오 소스 컴포넌트는 서로 다른 사운드 에셋을 사용한다.

개발자의 재량에 따라 새로운 유형의 오디오 소스 컴포넌트가 필요한 경우, 직접 오디오 소스를 담을 변수, 오디오 소스에 매개변수인 사운드 클립을 출력시킬 메서드를 스크립트에 구현하고, 인스펙터에서 오디오 소스 컴포넌트를 추가하도록 한다.

### 사운드 매니저에서 구현되어야 하는 것

- 싱글톤 패턴
- 각 `Audio Source`에 매개변수 `AudioClip`을 출력시키는 메서드

## 오디오 믹서

`AudioMixer`는 논리적 그룹의 사운드 크기를 관리한다.
사운드의 종류는 다음과 같다:

| 항목     | 설명                                                                   |
| ------ | -------------------------------------------------------------------- |
| Master | 전체 볼륨을 조절한다.                                                         |
| SFX    | 플레이어 사운드, 투사체 사운드, 공격 사운드, 적 사운드 등 게임 속에서 등장하는 사운드 이펙트의 소리 크기를 조절한다. |
| BGM    | 게임의 배경음악의 볼륨 크기를 조절한다.                                               |
| UI     | UI 클릭 사운드 크기를 조절한다.                                                  |
